SWF Activity Module Service Library

This directory contains scripts that give Flash applications access to Moodle lib functions.

Please note: These are not yet working scripts. The $swfId and $instance variables have been hard coded to make testing in the AMFPHP service browser possible. These variables identify the SWF Activity Module instance so that Moodle can check if the current user has the necessary capabilities to execute the class methods.

To use the service browser:

  1. In start up your development server and login to your development Moodle (don't do this on a public server!)
  2. On a course, create an instance of the SWF Activity Module with any Flash application embedded, and make a note of the instance ID, i.e. http://localhost/moodle/mod/swf/view.php?id=10 and the module ID, i.e. look at the module instance page source and find swfId=4 for example.
  3. In each of the service classes in http://localhost/moodle/lib/amfphp/services/, change these lines: $obj['instance'] = 76; and $obj['swfId'] = 4; In some cases, such as Groups.php, it is necessary to change more variables and create dummy users and groups to test the services but you get the basic idea.
  4. Go to http://localhost/moodle/lib/amfphp/browser/ - The Flex service browser will initialise and automatically load all the classes in the services directory.
  5. In the service browser, you can call all the class methods in the service library and see the returned results.
